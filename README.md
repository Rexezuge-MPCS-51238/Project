# AWS AccessBridge

Week 1 establishes the Cloudflare Workers foundation for a self-hosted AWS multi-account access portal.

Included scope:

- Next.js/OpenNext shell for Cloudflare Workers.
- Wrangler template for D1, KV, and Cloudflare Secrets Store bindings.
- Cloudflare Zero Trust user extraction, HMAC validation helpers, and AES-GCM credential encryption.
- D1 migrations and DAOs for credentials, AWS account nicknames, user metadata, and assumable-role primitives.
- Admin setup flow for Account, Credentials, and Chain onboarding.

The end-user role dashboard, grant management, governance features, and analytics are added in later weekly commits.
